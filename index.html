Principles

Evidence-Based Principles

Spaced Repetition

Distribute practice over time. 3×20 min across a week = 2× better retention than a single 1-hour block.

Retrieval Practice

Testing IS learning. Frequent low-stakes quizzes beat re-reading every time.

Interleaving

Mix problem types. Feels harder, produces deeper and more transferable understanding.

Cognitive Load Management

Reduce extraneous load (bad UI). Increase germane load (productive struggle, worked examples).

Desirable Difficulties

Making learning slightly harder in the right ways dramatically improves long-term retention.

Dual Coding

Words + diagrams > words alone. But don't narrate what's already on screen.

The Secret Weapon

Knowledge Graphs × Learning

Model knowledge as a connected graph, not a flat list of topics.

What a Learning Knowledge Graph Does

  • Maps prerequisites & dependencies between concepts
  • Identifies optimal learning sequences
  • Reveals knowledge gaps by tracing missing connections
  • Enables "if you know X, you're ready for Y" logic
  • Powers truly adaptive pathways — not just branching

Example: Data Science Curriculum

[Statistics] → [Probability] → [ML Foundations]
[Python] → [Pandas/NumPy] ↗
[Linear Algebra] → [Optimization] ↗

Learner knows Python + Statistics?
→ Skip basics, surface Probability + Pandas next
→ Personalized. Efficient. Evidence-based.

The Analogy

A learning platform without a knowledge graph is a GPS with no map.

It knows where you want to go, but it's guessing the roads.

A knowledge graph builds the map. Every concept. Every prerequisite. Every connection.

The AI navigates the student through the actual structure of the domain.

Live Example

What a Domain Knowledge Graph Looks Like

An "Intro to Machine Learning" domain graph — structured for adaptive learning:

--- domain: Introduction to Machine Learning entities: 38 relationships: 94 target_learner: Undergraduate, no prior ML --- ## Supervised Learning - Classification → prerequisite: Linear Algebra (vectors) → prerequisite: Probability (Bayes) → teaches: Decision boundaries, loss functions → leads_to: Neural Networks → misconception: "More data = better model" - Regression → prerequisite: Statistics (mean, variance) → prerequisite: Calculus (gradient) → teaches: Cost functions, gradient descent → leads_to: Regularization (L1, L2) ## Model Evaluation - Overfitting vs Underfitting → prerequisite: Supervised Learning → teaches: Bias-variance tradeoff → leads_to: Cross-validation

Learning Path Rules

IF learner.completed(Linear Algebra)
  AND learner.completed(Probability)
THEN → unlock: Classification

IF learner.score(Classification) < 70%
THEN → remediate: Probability (Bayes)

IF learner.completed(Supervised)
  AND learner.completed(Unsupervised)
THEN → unlock: Model Evaluation

What This Enables

The AI doesn't just quiz students — it knows the shape of the domain. It knows Classification requires Linear Algebra. A student struggling with K-Means probably has a gap in distance metrics. "More data = better" is a misconception to actively correct. The AI becomes a tutor that understands the subject.
